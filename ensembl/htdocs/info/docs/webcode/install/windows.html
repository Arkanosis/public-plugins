Additional Installation Instructions for Windows

Please note that Windows is not a supported platform - if you require further help, you may find it useful to contact the Ensembl developer community via our mailing list.

In all the following instructions, "Ensembl server root" refers to the directory where you are installing the Ensembl website software.

Installing software needed by Ensembl

If you do not already have Perl, CVS and BioPerl installed on your Windows system, you will need to install them first:

  1. Download/install activeperl (ActivePerl-5.8.8.820-MSWin32-x86-274739.msi)
  2. Install DBD::mysql
  3. ownload/install cvs (e.g., TortoiseCVS-1.8.31.msi)
  4. Add TortoiseCVS directory to "path" system variable
  5. In your Ensembl server root, make directory bioperl-live
  6. At the command prompt: 
    cvs -d :pserver:cvs@code.open-bio.org:/home/repository/bioperl login
    Then enter the password: cvs
  7. cd bioperl-live
cvs -d :pserver:cvs@code.open-bio.org:/home/repository/bioperl checkout -r bioperl-release-1-2-3 bioperl-live

Installing the Ensembl software

  1. In your Ensembl server root, make directories ensembl and enseml-variation
  2. At the command prompt: 
    cvs -d :pserver:cvsuser@cvs.sanger.ac.uk:/cvsroot/ensembl login
    Then enter the password: CVSUSER
  3. cd ensembl
cvs -d :pserver:cvsuser@cvs.sanger.ac.uk:/cvsroot/ensembl checkout -r branch-ensembl-[[SPECIESDEFS::ENSEMBL_VERSION]] ensembl
cd ensembl-variation
cvs -d :pserver:cvsuser@cvs.sanger.ac.uk:/cvsroot/ensembl checkout -r branch-ensembl-[[SPECIESDEFS::ENSEMBL_VERSION]] ensembl-variation
  4. Define system variable PERL5LIB=<server root>\bioperl-live;<server root>:\ensembl\modules;<server root>\ensembl-variation\modules
  5. If existing mysql-driver ("libmysql.dll") doesn't work: replace it, for example, by the one distributed with "php-5-2-3-win32- installer.msi"

Updating your Windows checkout

At the command prompt:

cd ensembl
cvs -q update -d -P -r branch-ensembl-[[SPECIESDEFS::ENSEMBL_VERSION]] ensembl
cd ensembl-variation
cvs -q update -d -P -r branch-ensembl-[[SPECIESDEFS::ENSEMBL_VERSION]] ensembl-variation
