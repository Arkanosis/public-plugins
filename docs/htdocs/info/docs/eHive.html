eHive production system

Introduction

This is a distributed processing system based on 'autonomous agents' and the behavioural structure of honey bees. It implements all functionality of both data-flow graphs and block-branch diagrams which should allow it to codify any program, algorithm, or parallel processing job control system. It is not bound to any processing 'farm' system and can be adapted to any GRID.

A pipeline is defined by a set of initial jobs and a graph whose nodes are analyses ("types" of jobs) and whose arcs represent dataflow rules (rules that control how running jobs generate new jobs) and analysis control rules (rules that block jobs of one analysis until all jobs of another analysis have completed).

Documentation

eHive is now hosted on GitHub, like the other Ensembl components, and the up-to-date documentation is to be found there. We currently maintain three stable (they only receive bug-fixes) versions of eHive.

  • v2.1 (recommended): introduces a better tracking of workers that specialize in multiple tasks.
  • v2.0: the first version to be independent from the Ensembl Core API.
  • v1.9: still dependent on the Ensembl Core API (for legacy only).
